Use this only when the user explicitly asks to generate values from test results.
Terminology note:
- "Generator" means a value-generation strategy under `GenerationStrategies` (test-driven), not a dedicated generator flow block type.

Purpose of generators:
- Derive a property value from test input/output evidence.
- Run when required test execution fails.
- Write the generated value to the configured target property (one-shot assignment).

Preconditions:
- Required test cases exist and are marked `RequiredForExecution=true`.
- A compatible generation strategy exists in the target flow block's `GenerationStrategies`.
- Input field and target property are configured.
- At the target flow block, define at least two expectations: one value expectation and one dataset-count expectation.

Setup order:
0. Discover available generator types when unclear:
   - Call `GetSupportedTypes` with `baseTypeFullName="FlowBlox.Core.Models.FlowBlocks.Additions.FlowBloxGenerationStrategyBase"`.
   - Inspect selected strategy details with `GetTypeKindsInfo`.
1. Ensure test definitions exist and are connected/configured.
   Link test definitions via standard update logic, for example `UpdateFlowBlock` with `path=/TestDefinitions/Link` and value `{"resolveManagedObjectByName":"<testDefinitionName>"}`.
2. For runtime-start test execution / generator application, explicitly set `RequiredForExecution=true` on the used test definitions.
3. Create/update generation strategy on the target flow block via `UpdateFlowBlock` at `GenerationStrategies` (exact property name).
   Important: `GenerationStrategies` is a FlowBloxReactiveObject collection (not a ManagedObject reference list), so do not use `/GenerationStrategies/Link`.
   Use path-based creation/update with typed index when needed, e.g. `/GenerationStrategies/0:FlowBlox.Core.Models.FlowBlocks.AIRemote.AIPropertyValueGenerationStrategy/...`.
4. Before detailed updates, call kind/type info tools (for example `GetTypeKindsInfo`) to confirm valid paths/options.
5. Configure strategy details via `UpdateFlowBlock` (nested strategy path):
- provider,
- input field,
- target property,
- prompt/template options.
Hint for `AIPropertyValueGenerationStrategy` prompt design:
- If unsure how to structure the prompt with input value and test expectations, use `GetToolboxEntries` for toolbox category `AIPropertyValueGenerationPrompts` and adapt a fitting template.
- For `AIPropertyValueGenerationStrategy`, prefer toolbox templates first; do not invent prompt-template assumptions when a fitting toolbox template exists.
- For structured target properties (for example Start/End pattern lists), require JSON-only output and include `$GenerationStrategy::TargetPropertyDescription` in the prompt template.
6. Execute tests / runtime as requested by the user.

Minimum expectations at target flow block:
- Value expectation: target `FirstValue` or `AnyValue`, operator `ComparisonOperator.Equals`, expected value provided.
- Tip: Instead of `AnyValue`, prefer selection mode `UserInput_ExpectedValue` when downstream flow blocks should continue execution, because it combines selection and expectation.
- Dataset-count expectation: target `NumberOfDatasets`, operator `ComparisonOperator.Equals`, expected count provided.
- More expectation conditions usually improve generator quality.

Minimum test-case requirements:
- `AIPropertyValueGenerationStrategy`: at least 1 suitable test case.
- `SequenceDetectionGenerationStrategy`: at least 2 automatically executable test cases.

Use-Cases / Examples:
1. Happy path (simple target property):
- Create `AIPropertyValueGenerationStrategy` on the target flow block.
- Link/select the strategy input field.
- Use a fitting prompt template from `GetToolboxEntries` (for example Regex/XPath template) and set target property.
- Run tests/generation; generated value is assigned to the simple target property.
2. Special use-case (complex/structured target property):
- Example: `StartEndPatternFlowBlock.StartEndPatterns`.
- Provide $GenerationStrategy::TargetPropertyDescription in the prompt so the AI sees the full target schema.
- Convenience: You can also use the toolbox template `Start/End Pattern JSON Generator`.
- Require JSON-only output; AI returns JSON matching the schema.
- Generator value can then be deserialized into the target property type and assigned.

Do not create or wire generators unless the user explicitly requests it.
